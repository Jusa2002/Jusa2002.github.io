JSON Serializer API

A dependency-free Kotlin library for converting annotated objects to JSON.

Overview

JSON is the standard format for data exchange on the web. It's a tree-shaped structure, which works well for most cases — but becomes problematic when the data you want to serialise is a graph, where objects reference each other. A naive serialisation would either duplicate shared objects or fall into infinite recursion. The standard workaround is to assign each object a UUID and replace back-references with a special {"$ref": "...uuid..."} object. But managing those UUIDs manually is tedious and error-prone.

ProJson is a Kotlin library built to solve exactly that — making UUID-based reference management fully transparent to the caller. The library was built from scratch without any external dependencies, using Kotlin reflection to inspect objects at runtime and convert them into a structured in-memory JSON model (JsonObject, JsonArray, JsonPrimitive, JsonReference).

The library was developed in three phases. The first covered standard JSON serialisation — converting any Kotlin object or collection into JSON, with support for in-memory manipulation (add, modify, delete properties). The second phase introduced annotation-driven customisation: a @Reference annotation to mark properties that should use UUID references instead of inline objects; @JsonProperty to rename fields in the output; @JsonIgnore to exclude fields entirely; and a plugin mechanism via @JsonString for custom string serialisation of specific types. The third phase focused on documentation, publishing the library to GitHub with a JAR release, and writing a full usage tutorial in the README.

The project was part of the Advanced Programming course at ISCTE's MSc in Computer Science, and was a strong exercise in API design, reflection, and building developer-facing abstractions.

Tech Stack

Kotlin · JVM Reflection · Custom Annotations · API Design · No External Libraries
