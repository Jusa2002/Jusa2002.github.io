Biocomputation — Hamiltonian Paths

Graph search inspired by DNA strand construction and destruction.

Overview

In 1994, Leonard Adleman demonstrated that biological molecules could be used to solve computational problems, specifically the Hamiltonian Path Problem (HPP) — the challenge of finding a path through a directed graph that visits every vertex exactly once. His key insight was that DNA hybridisation naturally explores all possible paths in parallel, something conventional computers struggle to do efficiently.

This project implements a discrete stochastic simulation of that experiment in Python. Each edge of the input graph is represented as an individual path fragment, randomly distributed in a 3D unit cube. The simulation then evolves through three types of events: concatenation (two compatible path fragments merge into one), displacement (fragments move randomly within the cube, with shorter paths moving further), and cleavage (paths that have grown too long are broken into shorter fragments). This mirrors the biochemical steps of hybridisation, PCR selection, and gel electrophoresis from the original experiment.

After the simulation runs for a set time limit, a filtering stage eliminates any paths that don't start and end at the required vertices, don't have the right length, or skip any vertex. Whatever survives is a valid Hamiltonian path. The program was validated on two graphs, including the original 7-node graph used by Adleman himself.

The project was structured following a modular, layered programming methodology — first defining the core data types (graphs, paths, individuals, events), then building the simulation logic on top of them.

Tech Stack

Python · Discrete Stochastic Simulation · Graph Theory · Biocomputation (Adleman's Model)
